The header does not have a position so therefore it is static. When it's position is static that means that it is in it's regular position of the documents flow. Along with the position being static, we have not changed any other properties such as left, right, top, and bottom. So when it comes to the header for the holy grail, it's position is not messed with nor do we mess with any property associated with it so it just goes with the document's flow.

Unlike the header, the navigation bar has fixed position. This means that it is fixed to wherever it's been placed at. For example, if you were making a website and wanted to keep the navigation bar visible at all times no matter where you scroll the page then you would use fixed positioning. Along with the position property we also changed the top and left property. This means that with the values we had put in the nav bar will that amount of pixels away from the top and left of the page.

When analyzing the article element we can see that the position is relative. This means that the element that is being placed is slightly positioned offset from where the normal position would be once it's other position elements are changed as well. If the other position elements are not changed then it's positing won't matter; however, in this example we have changed the top 15 pixels so it will be moved 15 pixels from the top of the container that it is currently in.

The aside element's position has been listed as absolute. This means that this specfic element has now been removed from the flow of the document and now seemingly exists independently of this document. Along with it having a position of absolute, it also has had the top element changed so that way it is now 235 pixels away from the top of the page.

Just like the article's position, the footer also has a fixed position. This means that this specific element is now fixed to the spot it is currently placed in and even if I scroll the page it won't move. Along with having a fixed position, the bottom element has also been changed to 0 pixels meaning that now the footer is on the bottom on the page.
